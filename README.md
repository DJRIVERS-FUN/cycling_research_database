# Cycling Research Database

A research-oriented workflow for combining Strava activity metadata, Garmin FIT/GPX data, and Di2stats shifting data into a clean, reproducible cycling database.

## Purpose

This repository stores the code, documentation, schemas, and website-ready exports for a cycling research database focused on human-machine performance regulation, gearing behaviour, terrain, and ride context.

## Data workflow

The project uses three data layers:

1. **Raw data** — original files downloaded from Strava, Garmin, and Di2stats. Raw files should usually be kept locally and not committed to GitHub.
2. **Processed data** — cleaned CSV/Parquet/SQLite outputs generated by Python.
3. **Website data** — lightweight JSON files for live dashboards and GitHub Pages.

## Recommended local structure

```text
cycling_research_database/
  raw/

    di2stats/
  processed/
  database/
  website_json/
  scripts/
  schema/
  docs/
  notebooks/
```

## Repository structure

```text
scripts/        Python processing scripts
schema/         Database and column definitions
docs/           Workflow documentation
website_json/   Lightweight JSON files for website dashboards
website/        Static website/dashboard files
notebooks/      Exploratory analysis notebooks
```

## Core principle

Keep raw data untouched. Generate all processed and website-facing files reproducibly from scripts.
